# ImageAnalysis with Vision Language Models

Sample notebooks that use vision-capable LLMs (Claude Opus 4.x / GPT-5.x) to detect **missing dimensions** in engineering drawings by comparing a baseline drawing against a check drawing, and annotating the omissions with red circles on the check drawing.

The notebooks demonstrate three different execution styles against the same task:

| Notebook | Backend | Approach |
|---|---|---|
| [01-diagram_check01-gpt5_4.ipynb](01-diagram_check01-gpt5_4.ipynb) | Azure AI Foundry (OpenAI Responses API) | Single-shot `responses.create` with `code_interpreter` and image inputs as base64 |
| [02-diagram_check01-opus4_8.ipynb](02-diagram_check01-opus4_8.ipynb) | Azure AI Foundry (Anthropic) | Multi-turn streaming with a **checkpoint** file inside the container, plus a custom Skill |
| [02-managed-agent02.ipynb](02-managed-agent02.ipynb) | Public Anthropic API (Managed Agents) | Server-side agent loop using `beta.agents` / `environments` / `sessions` with a custom Skill |

The custom skill used by the Anthropic notebooks lives in [.claude/skills/missing-dimension-check/SKILL.md](.claude/skills/missing-dimension-check/SKILL.md).

## Prerequisites

- Python 3.10+
- One of the following, depending on which notebook you want to run:
  - **Azure AI Foundry** deployment exposing the Anthropic / OpenAI endpoints (for `01-…`, `02-diagram_check01-…`)
  - **Public Anthropic API key** with Managed Agents / Skills / Files beta access (for `02-managed-agent02.ipynb`)

## Setup

```bash
git clone https://github.com/<your-account>/ImageAnalysis-vision-language-models.git
cd ImageAnalysis-vision-language-models

python -m venv .venv
# Windows
.venv\Scripts\activate
# macOS / Linux
# source .venv/bin/activate

pip install -r requirements.txt
```

### Configure environment variables

Copy `sample.env` to `.env` and fill in the values you actually need:

```bash
cp sample.env .env
```

| Variable | Used by | Notes |
|---|---|---|
| `ANTHROPIC_ENDPOINT` | `02-diagram_check01-opus4_8.ipynb` | Foundry Anthropic base URL |
| `ANTHROPIC_API_KEY` | `01-…`, `02-diagram_check01-…` | Foundry API key |
| `CLAUDE_MODEL` | `02-diagram_check01-opus4_8.ipynb` | e.g. `claude-opus-4-8` |
| `OPENAI_ENDPOINT` | `01-diagram_check01-gpt5_4.ipynb` | Foundry OpenAI base URL |
| `OPENAI_MODEL` | `01-diagram_check01-gpt5_4.ipynb` | e.g. `gpt-5.5` |
| `CLAUDE_API_KEY` | `02-managed-agent02.ipynb` | Public Anthropic API key |
| `MODEL` | `02-managed-agent02.ipynb` | Model ID on the public API |

`.env` is git-ignored — see `.gitignore`.

### Add input drawings

Place your input images under `data/` (the directory is git-ignored so drawings are not accidentally published):

```
data/
├── sample01.png            # baseline drawing (fully dimensioned)
├── check01.png             # drawing to be checked
├── sample01_resized.png    # long-edge ≤ 1568 px (generated by the notebook)
└── check01_resized.png
```

The resized variants are created automatically by the resize cell in `02-diagram_check01-opus4_8.ipynb`.

## Running

Open any notebook in Jupyter / VS Code and run cells top-to-bottom.

The Anthropic notebooks write generated images (annotated drawings, checkpoints, etc.) into `./out/`, which is git-ignored.

### Notes on the notebooks

- **`02-diagram_check01-opus4_8.ipynb`** hard-codes some `file_…` IDs in the "Retrieve uploaded files" cell. These are placeholders — replace them with the IDs printed by your own upload cell, or just re-run the upload cell to get fresh IDs.
- **`02-managed-agent02.ipynb`** hard-codes an `ENVIRONMENT_ID` and two `file_…` IDs so subsequent runs can reuse an existing environment / files. Comment out those cells and use the freshly-created objects when running for the first time.
